# 12. API リファレンス・エラーコード

マジカルジャーナルの REST API のエンドポイント仕様と、全エラーコードの一覧・意味・対処法をまとめる。運営がユーザーから「エラー画面に出たコード」を伝えられたとき、このドキュメントで原因と対処を特定できることを目的とする。

- **一次定義**: エラーコードのカタログは `packages/api/src/errors.ts`(API 側)と `packages/liff/src/errors.ts`(フロント側)。**コードを追加・変更したら必ず両方のカタログとこのドキュメントを更新すること。**
- **表示形式**: ユーザーに表示されるエラーは `メッセージ(コード)` の形式(例: `該当する住所が見つかりませんでした(MJ-POST-002)`)。
- **API 応答形式**: エラー時は HTTP ステータス + JSON `{ "error": "メッセージ", "code": "MJ-XXX-NNN" }`。500 系は詳細を隠し `MJ-SYS-001` を返す。

---

## エラーコードの読み方

`MJ-<領域>-<番号>`

| 領域 | 意味 | 発生層 |
|------|------|--------|
| `AUTH` | 認証・認可 | API |
| `USER` / `PROFILE` | ユーザー・プロフィール未登録 | API |
| `REG` | 新規登録 | API |
| `PROF` | プロフィール更新 | API |
| `POST` | 郵便番号検索 | API |
| `SYS` | サーバー内部 | API |
| `NET` | クライアント↔サーバー通信 | フロント |
| `MAP` | 地図・住所ジオコーディング | フロント |
| `LIFF` | LIFF(LINE ログイン)初期化 | フロント |

---

## エラーコード一覧

### 認証・認可(AUTH)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-AUTH-001` | 401 | 認証情報がありません | Authorization ヘッダーが無い/Bearer 形式でない | LIFF から正しく起動しているか確認。LIFF エンドポイント URL 設定漏れの可能性 |
| `MJ-AUTH-002` | 401 | 開発用トークンの形式が不正です | 開発モードで `dev:` の後の ID が空 | 開発時のみ。`Bearer dev:<userId>` 形式で送る |
| `MJ-AUTH-003` | 401 | 認証トークンの検証に失敗しました | LINE の ID トークン検証 API が失敗(期限切れ・チャネル不一致) | `LINE_LOGIN_CHANNEL_ID` が LIFF のチャネルと一致しているか確認。トークン期限切れなら再ログイン |
| `MJ-AUTH-004` | 403 | このアカウントは利用が許可されていません | 許可リスト(`ALLOWED_LINE_USER_IDS`)外のユーザー | クローズド運用の仕様。招待する場合は `.env` の許可リストにユーザー ID を追加 |

### ユーザー・プロフィール未登録(USER / PROFILE)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-USER-404` | 404 | ユーザーが登録されていません | 認証は通ったがユーザー未登録 | 正常系。フロントは登録画面へ誘導する。異常なら DB の users テーブルを確認 |
| `MJ-PROFILE-404` | 404 | プロフィールが登録されていません | ユーザーは在るがプロフィール未作成 | 通常は起きない(登録時に必ず作成)。DB の profiles テーブルを確認 |

### 新規登録(REG)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-REG-001` | 400 | 必須項目が入力されていません | birthDate / nameKana / nameRomaji / charStyle のいずれか欠落 | フロントのバリデーションで防ぐ想定。直叩き時はペイロード確認 |
| `MJ-REG-002` | 400 | 表示スタイルの指定が不正です | charStyle が male/female 以外 | フロントの選択肢を確認 |
| `MJ-REG-003` | 400 | 生年月日の形式が不正です | birthDate が YYYY-MM-DD でない | 入力値を確認 |
| `MJ-REG-004` | 400 | 出生時刻の形式が不正です | birthTime が HH:MM でない | 入力値を確認 |
| `MJ-REG-409` | 409 | すでに登録済みです | 登録済みユーザーが再登録を試行 | 正常系。フロントは今日のページへ誘導する |

### プロフィール更新(PROF)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-PROF-001` | 400 | 表示スタイルの指定が不正です | charStyle が male/female 以外 | 入力値を確認 |
| `MJ-PROF-002` | 400 | 出生時刻の形式が不正です | birthTime が HH:MM でない | 入力値を確認 |

### 郵便番号検索(POST)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-POST-001` | 400 | 郵便番号は7桁で入力してください | 郵便番号が7桁でない | 数字7桁で入力(ハイフンは自動除去) |
| `MJ-POST-002` | 404 | 該当する住所が見つかりませんでした | その郵便番号の住所が存在しない | 郵便番号を確認。または住所を直接入力してもらう |
| `MJ-POST-003` | 502 | 郵便番号検索サービスでエラーが発生しました | 上流 API が 4xx/5xx を返した | 上流(jp-postal-code-api)の障害。時間をおいて再試行、または住所直接入力で回避 |
| `MJ-POST-004` | 502 | 郵便番号検索サービスに接続できませんでした | 上流 API へ接続不可(ネットワーク・DNS) | API サーバー(NAS)の外部通信を確認。上流障害の可能性。住所直接入力で回避 |

### サーバー内部(SYS)

| コード | HTTP | メッセージ | 意味 | 対処法 |
|--------|------|-----------|------|--------|
| `MJ-SYS-001` | 500 | サーバー内部エラーが発生しました | 未捕捉の例外(DB エラー等)。詳細はクライアントに露出しない | **サーバーログを確認**(`docker compose logs api`)。`[API Error]` 行に原因メッセージが出る |

### 通信(NET・フロント)

| コード | メッセージ | 意味 | 対処法 |
|--------|-----------|------|--------|
| `MJ-NET-001` | サーバーとの通信に失敗しました | API が構造化されていない応答を返した/通信層で失敗 | API の稼働・URL(`VITE_API_BASE_URL`)・CORS 設定を確認 |

### 地図・住所(MAP・フロント)

| コード | メッセージ | 意味 | 対処法 |
|--------|-----------|------|--------|
| `MJ-MAP-001` | 地図サービスのAPIキーが設定されていません | 本番で `VITE_GOOGLE_MAPS_API_KEY` 未設定 | Cloudflare のビルド環境変数に Maps API キーを設定して再デプロイ |
| `MJ-MAP-002` | 住所から位置を特定できませんでした… | ジオコーディングが結果なし/タイムアウト | 住所を市区町村から見直す。Maps キーのリファラー制限も確認(`RefererNotAllowedMapError`) |
| `MJ-MAP-003` | 地図の読み込みに失敗しました | 地図(方位マップ)の初期化失敗 | Maps キー・リファラー制限・ネットワークを確認 |

### LIFF 初期化(LIFF・フロント)

| コード | メッセージ | 意味 | 対処法 |
|--------|-----------|------|--------|
| `MJ-LIFF-001` | LINEログイン情報の取得に失敗しました | LIFF SDK 未読込 or `VITE_LIFF_ID` 未設定 | `VITE_LIFF_ID` の設定・LIFF SDK の読込・LIFF アプリのエンドポイント URL を確認 |

## API エンドポイント リファレンス

ベース URL: 本番 `https://mj-api.iterra.online` / ローカル `http://localhost:3000`

- **認証**: `/api/health` を除く全 `/api/*` は `Authorization: Bearer <IDトークン>` 必須。開発モードは `Bearer dev:<userId>`。
- **CORS**: `*.pages.dev` / `*.workers.dev` / `localhost` および `ALLOWED_ORIGINS` で許可したオリジンのみ。

### GET /api/health
ヘルスチェック(認証不要)。`200 { "status": "ok" }`。

### POST /api/register
新規登録し、enabled モジュールを全診断して保存する。

リクエスト:
```json
{
  "birthDate": "1990-05-17",        // 必須 YYYY-MM-DD
  "birthTime": "08:30",             // 任意 HH:MM
  "nameKana": "やまだ たろう",       // 必須
  "nameRomaji": "YAMADA TARO",      // 必須(ヘボン式)
  "addressText": "東京都新宿区",     // 任意
  "lat": 35.6938, "lng": 139.7034,  // 任意(方位マップ中心)
  "charStyle": "male"               // 必須 male|female
}
```
応答: `201 { "message": "Registration successful", "userId": 1 }`
主なエラー: `MJ-REG-001..004`, `MJ-REG-409`

### GET /api/today
今日の日盤・方位判定・当日運勢を返す。応答に `homeLatLng`(方位マップ中心)、`dayBan`、`directions.{day,month,year}`、`fortune` を含む。
主なエラー: `MJ-USER-404`, `MJ-PROFILE-404`

### GET /api/profile
プロフィール + 全診断結果(`diagnosis[]`)を返す。
主なエラー: `MJ-USER-404`, `MJ-PROFILE-404`

### PATCH /api/profile
プロフィールを更新する。`birthTime` 変更時はポテンシャルタイプを再診断する。

リクエスト(すべて任意):
```json
{ "birthTime": "23:30", "addressText": "...", "lat": 0, "lng": 0, "charStyle": "female" }
```
主なエラー: `MJ-USER-404`, `MJ-PROF-001`, `MJ-PROF-002`

### GET /api/diagnosis
診断結果をモジュール別に返す。
主なエラー: `MJ-USER-404`

### GET /api/postal?zipcode=NNNNNNN
郵便番号 → 住所を返す(GitHub Pages ホストの静的 API をサーバー側プロキシ。第三者へ渡すのは郵便番号のみ)。

応答:
```json
{ "zipcode":"1500001", "prefecture":"東京都", "city":"渋谷区", "town":"神宮前", "address":"東京都渋谷区神宮前" }
```
主なエラー: `MJ-POST-001..004`

## 運営向け: 障害切り分けの早見表

| ユーザーが見たコード | まず疑う場所 |
|--------------------|-------------|
| `MJ-AUTH-*` | LIFF/LINE Login の設定、許可リスト(`.env`) |
| `MJ-SYS-001` | `docker compose logs api` のサーバーログ |
| `MJ-POST-003/004` | 上流の郵便番号 API 障害、NAS の外部通信 |
| `MJ-MAP-*` / `MJ-NET-001` | Cloudflare 環境変数(Maps キー・API URL)、Maps のリファラー制限 |
| `MJ-USER-404` / `MJ-REG-409` | 正常系(登録有無による分岐)。異常時のみ DB 確認 |
